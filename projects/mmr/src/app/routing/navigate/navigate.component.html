Outlets

HOVER over LINKS to see the suggested URL's

"['',{outlets: {namedOutlet: 'outlet'}}] | Current page Works
['/',{outlets: {namedOutlet: 'outlet'}}] | Current page Works
['/navigate',{outlets: {namedOutlet: 'outlet'}}] Not Works (also below)
['/params',{outlets: {namedOutlet: 'outlet'}}] | NOTE: Navigating to a new page adds a slash b4 the outlet and may require a different app.routing path or a second navigation call based on a parameter.
[{outlets: {namedOutlet: null}}] | Close Outlet
[{outlets: {namedOutlet: null}}] [queryParams]="{analytics:300}" | Close with params

Lessons::

  1. You cannot navigate to a new component and specify an outlet (unless there is something I don't know yet)
  2. Workaraound: Navigate with query params and then navigate the outlet. ( Be aware that you want to have urls with States in mind )
  3. You can have multiple outlets with params as well, google multiple router outlets stackblitz. chinese one

Navigate to Outlet

this.myRouter.navigate([ ' ',{outlets:{namedOutlet: 'outlet'}}])
this.myRouter.navigate(['/',{outlets:{namedOutlet: 'outlet'}}])/navigate(namedOutlet:outlet) Same result for both

Note::...

If you have nested router-outlets you would have to use child routes in app.routing as per examples.
You can have multiple named routers side by side. as per chinese version. It was using child routes, no primary routes.
If navigating to another router-outlet you may have to navigate via slashes
